## Data Protection

- **Data Protection:** Controls and processes used to keep data confidential, accurate, available, and properly handled.
- Data protection applies to files, databases, emails, backups, cloud storage, logs, and printed documents.

## Data States

- **Data at Rest:** Data stored on disk, database, backup, or cloud storage.
  - Example: File stored on a server.

- **Data in Transit:** Data moving across a network.
  - Example: HTTPS web traffic.

- **Data in Use:** Data actively being processed by a system or user.
  - Example: Open document in memory.

## Data Classification

- **Data Classification:** Labeling data based on sensitivity and required protection.

Examples:
- Public
- Internal
- Confidential
- Restricted

Classification helps determine:
- Who can access data
- How data should be stored
- How data should be transmitted
- How data should be disposed of

## Data Handling

Data handling rules may include:
- Store sensitive data only in approved locations
- Encrypt sensitive files
- Restrict access by role
- Do not share confidential data through personal email
- Use approved cloud services
- Follow retention requirements

## Data Loss Prevention

- **DLP:** Data Loss Prevention technology used to detect or prevent unauthorized data movement.

DLP can monitor:
- Email
- Endpoint copy activity
- Cloud uploads
- Web traffic
- Removable media

Example:
- DLP blocks a user from emailing a spreadsheet containing credit card numbers outside the company.

## Encryption and Masking

- **Encryption:** Converts readable data into unreadable form without the correct key.
- **Masking:** Hides part of the data while keeping it usable.
- **Tokenization:** Replaces sensitive data with a non-sensitive token.

Examples:
- Encrypt laptop hard drive
- Mask credit card number except last four digits
- Tokenize payment card data

## Retention and Disposal

- **Retention:** Defines how long data should be kept.
- **Disposal:** Secure removal of data when it is no longer needed.

Disposal methods:
- Shredding
- Wiping
- Degaussing
- Physical destruction
- Secure cloud deletion process

## Privacy Data

- **PII:** Personally Identifiable Information.
- **PHI:** Protected Health Information.
- **SPII:** Sensitive Personally Identifiable Information.

Examples:
- Name, address, phone number
- Social Security number
- Medical records
- Financial account information

## Key Takeaways

- Data must be protected based on sensitivity and business value.
- Classification, encryption, DLP, retention, and secure disposal all support data protection.
