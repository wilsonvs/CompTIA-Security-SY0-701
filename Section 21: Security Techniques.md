## Security Techniques

- **Security Technique:** A method or control used to protect systems, detect threats, reduce risk, or recover from incidents.
- Security techniques work best when layered together.

## Preventive Controls

- **Preventive Control:** Stops or reduces the chance of an incident before it happens.

Examples:
- MFA
- Firewall rule
- Least privilege
- Encryption
- Patch management
- Security awareness training
- Secure configuration baseline

## Detective Controls

- **Detective Control:** Identifies suspicious or unwanted activity.

Examples:
- SIEM alert
- IDS alert
- EDR alert
- File integrity monitoring
- Log review
- Vulnerability scan
- User behavior analytics

## Corrective Controls

- **Corrective Control:** Fixes a problem or reduces damage after an event.

Examples:
- Patch deployment
- Malware removal
- Account reset
- Backup restoration
- Configuration rollback
- Firewall block rule

## Compensating Controls

- **Compensating Control:** Alternate control used when the preferred control cannot be used immediately.

Example:
- If a patch cannot be installed yet, restrict access with firewall rules and increase monitoring.

## Cryptographic Techniques

- **Hashing:** Verifies data integrity.
- **Encryption:** Protects confidentiality.
- **Digital Signature:** Supports integrity, authentication, and non-repudiation.
- **Certificate:** Supports trust and secure communication.

## Operational Techniques

- Change management
- Access reviews
- Incident response playbooks
- Backup testing
- Tabletop exercises
- Security monitoring
- User training

## Key Takeaways

- Security techniques should support prevention, detection, response, and recovery.
- Layered controls reduce the chance that one failure causes a major incident.
