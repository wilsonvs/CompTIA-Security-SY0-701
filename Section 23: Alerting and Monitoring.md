# Alerting and Monitoring

## Objectives
- Understand how monitoring supports threat detection
- Identify important log sources
- Explain alert quality and tuning

## Monitoring
- **Monitoring:** Collecting and reviewing system, network, application, cloud, and security activity
- Monitoring helps detect suspicious behavior, policy violations, failures, and attacks

## Alerting
- **Alert:** A notification that specific activity may need review
- Alerts can be generated by SIEM, EDR, IDS/IPS, firewalls, cloud tools, or email security platforms

## Common Log Sources
- Authentication logs
- Firewall logs
- DNS logs
- VPN logs
- Proxy logs
- Endpoint logs
- Cloud audit logs
- Email security logs
- Web server logs

## Common Alert Examples
- Multiple failed logons
- Successful login after repeated failures
- Impossible travel
- New administrator account
- Malware detection
- Suspicious PowerShell command
- Public cloud storage change
- Unusual outbound traffic

## Alert Tuning
- **Tuning:** Adjusting alert logic to reduce false positives and improve accuracy
- Tuning may use:
  - Thresholds
  - Allow lists
  - Suppression windows
  - Asset criticality
  - User context
  - Correlation with other events

## Key Takeaway
- Monitoring provides visibility; alerting turns important activity into work that analysts can investigate.
