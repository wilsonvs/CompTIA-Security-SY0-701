# Incident Response

## Objectives
- Understand the incident response lifecycle
- Identify common incident response actions
- Explain why preparation and documentation are important

## Incident Response
- **Incident Response:** The process of preparing for, detecting, analyzing, containing, eradicating, and recovering from security incidents
- A structured process reduces confusion during high-pressure events

## Incident Response Lifecycle
1. **Preparation:** Build playbooks, tools, contacts, access, training, and logging
2. **Detection and Analysis:** Identify what happened and whether it is an incident
3. **Containment:** Limit damage and stop spread
4. **Eradication:** Remove the cause of the incident
5. **Recovery:** Restore systems and monitor for recurrence
6. **Lessons Learned:** Review what happened and improve controls

## Common Incident Types
- Phishing
- Malware infection
- Ransomware
- Account compromise
- Data exposure
- Unauthorized access
- Denial of service
- Lost or stolen device

## Common Response Actions
- Disable user account
- Reset password
- Isolate endpoint
- Block IP or domain
- Preserve logs and evidence
- Remove malware
- Patch exploited vulnerability
- Restore from backup

## Communication
- Incident communication should be clear, accurate, and timely
- Communication may involve IT, security, management, legal, HR, vendors, and affected users

## Key Takeaway
- Incident response is about controlling damage, preserving evidence, restoring service, and improving after the event.
